# Signal Studio

Программное средство для изучения методов цифровой обработки сигналов и проведения практических измерений. Разработано в рамках дисциплин, связанных с ЦОС.

---

## 1. Назначение

Signal Studio предназначен для лабораторного практикума и самостоятельной работы по цифровой обработке сигналов. В программе реализованы:

- генерация тестовых сигналов с настраиваемыми параметрами;
- частотно-избирательная фильтрация (ФНЧ, ФВЧ, полосовой фильтр) с выбором типа аппроксимации;
- спектральный анализ: амплитудный спектр и спектрограмма с настраиваемыми окнами;
- загрузка и сохранение аудиофайлов в формате WAV;
- режим работы в реальном времени с захватом сигнала с микрофона.

Интерфейс построен на PyQt6; для построения графиков используется PyQtGraph. Расчёты выполняются с помощью NumPy и SciPy.

---

## 2. Технические характеристики

### 2.1. Генератор сигналов

Реализованы одиннадцать типов сигналов: синусоидальный, треугольный, прямоугольный (с регулируемой скважностью), меандр, пилообразный, импульсный, постоянная составляющая (DC), линейный частотно-модулированный (ЛЧМ, chirp), трапецеидальный, гауссов импульс, белый шум.

Параметры генерации:

- частота дискретизации — выбирается из набора 8000, 11025, 16000, 22050, 32000, 44100, 48000, 96000 Гц;
- частота сигнала ограничена снизу и сверху (сверху — не более 0,99 от частоты Найквиста);
- амплитуда и постоянная составляющая (offset);
- начальная фаза (градусы);
- коэффициент заполнения (duty cycle) для прямоугольного и импульсного сигналов — в диапазоне 0,01…0,99;
- длительность; число отсчётов ограничено (не более 10⁷);
- для шума при необходимости задаётся фиксированный seed для воспроизводимости.

Формирование сигналов выполняется в `core/signal_processor.py` с использованием NumPy и функций SciPy (sawtooth, square, chirp и др.).

### 2.2. Цифровая фильтрация

Поддерживаются типы фильтров: без фильтра, ФНЧ, ФВЧ, полосовой. Реализация — каскад звеньев второго порядка (SOS) через `scipy.signal.sosfiltfilt` (нулевая фаза).

Методы аппроксимации: Баттерворт, Чебышёв I рода, Чебышёв II рода, эллиптический. Порядок фильтра от 1 до 10. Для Чебышёва I и эллиптического задаётся допустимая неравномерность в полосе пропускания (rp), для Чебышёва II и эллиптического — в полосе задерживания (rs). Частоты среза нормируются на половину частоты дискретизации и ограничиваются для устойчивости.

### 2.3. Спектральный анализ

Амплитудный спектр строится по одному сегменту (начало или конец сигнала). Используется одностороннее БПФ (rfft) с учётом коэффициента усиления оконной функции. Доступные окна: Ханна (Hann), Хэмминга, Блэкмана, Кайзер (beta=14). Длина сегмента (nperseg) ограничена размером сигнала; по умолчанию в коде используются значения порядка 2048–4096 отсчётов. Результат выводится в децибелах (20·lg(magnitude)); для пустого или нулевого сигнала показывается заглушка (−120 дБ).

Спектрограмма строится методом короткого времени Фурье: сегменты с перекрытием, оценка СПМ по сегментам, вывод в координатах время–частота в дБ. Параметры (nperseg, перекрытие) задаются в `core/fft.py`.

### 2.4. Работа с файлами WAV

Импорт: формат WAV PCM, разрядность 8, 16, 24 или 32 бита, моно или два канала. Частота дискретизации от 1 до 1 000 000 Гц. При стерео данные возвращаются в виде массива с двумя столбцами; при анализе можно использовать один канал или усреднение.

Экспорт: 16 бит, моно или стерео. Перед квантованием выполняется нормализация по пику (если максимум по модулю больше 1). Применяется дизеринг (равномерное распределение ±0,5 LSB) для уменьшения модуляции шума при слабых сигналах. Реализация — в `core/wav_handler.py` через стандартный модуль `wave`.

### 2.5. Режим реального времени

Захват аудио с устройства ввода по умолчанию через библиотеку sounddevice. Используется кольцевой буфер на 5 секунд при текущей частоте дискретизации устройства (при ошибке запроса принимается 44100 Гц). Размер блока ввода 1024 сэмпла. Обновление осциллограммы и спектральных оценок выполняется по таймеру. При отсутствии или недоступности устройства выводится предупреждение.

## 4. Теоретические основы

### 4.1. Теорема отсчётов

Сигнал с ограниченным спектром (0…F_max) однозначно восстанавливается по отсчётам, если частота дискретизации F_s ≥ 2·F_max. Частота F_s/2 называется частотой Найквиста; выше неё при заданной F_s спектр не воспроизводится. В программе частота сигнала ограничивается значением 0,99·F_s/2.

### 4.2. Односторонний амплитудный спектр

Для вещественного сигнала достаточно положительных частот. Пусть N — длина сегмента, W — оконная функция, G — среднее значение окна (коэффициент усиления). После БПФ сегмента x·W амплитуды нормируются: деление на N и на G. Компоненты, соответствующие положительным частотам (кроме нулевой и Найквиста), умножаются на 2; постоянная составляющая и компонента на частоте Найквиста не удваиваются. Результат переводится в дБ: 20·lg(magnitude).

### 4.3. Спектральная плотность мощности

Оценка СПМ по методу Уэлча: сигнал разбивается на сегменты (с перекрытием), к каждому применяется окно, вычисляется периодограмма, затем усреднение. В программе используется режим `scipy.signal.spectrogram` с масштабированием density. Результат в логарифмической шкале: 10·lg(PSD). Спектрограмма даёт зависимость этой оценки от времени и частоты.

## 5. Установка и запуск

Требуется Python 3.10 или новее.

Создание виртуального окружения и установка зависимостей (в корне проекта):

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Запуск:

```bash
python main.py
```

В `requirements.txt` указаны: PyQt6 (≥6.5.0), numpy (≥1.24.0), scipy (≥1.10.0), sounddevice (≥0.4.6), pyqtgraph (≥0.13.3).

## 6. Тестирование

Модульные тесты проверяют генерацию сигналов (в том числе граничные случаи и воспроизводимость шума по seed), применение фильтра, расчёт БПФ и спектрограммы, формирование кадров анализа, загрузку и сохранение WAV, обработку ошибок (несуществующий файл, пустые данные), а также функцию форматирования величин в конфиге. Запуск:

```bash
python -m pytest test_core.py -v
```

## 7. Сборка исполняемого файла (Windows)

В корне проекта выполнить:

```batch
build_exe.bat
```

Скрипт активирует виртуальное окружение, при необходимости обновляет PyInstaller, запускает сборку в один файл (onefile, windowed). Результат — `dist\Signal_Studio.exe`. Папка `hooks` передаётся в PyInstaller как дополнительный каталог с хуками; в ней задаётся хук для pyqtgraph, чтобы не подключать подмодули opengl, canvas и examples, из-за которых стандартная сборка завершалась с ошибкой.

## 8. Режимы работы и управление

В программе четыре основных режима, переключаемые из меню или сочетаниями клавиш:

| Режим              | Сочетание |
|--------------------|-----------|
| Генератор сигналов | Ctrl+1    |
| Анализатор WAV      | Ctrl+2    |
| Реальное время      | Ctrl+3    |
| Справочное руководство | Ctrl+4 |

Общие действия: Ctrl+O — открыть WAV, Ctrl+S — сохранить, Ctrl+Q — выход, F5 — применить параметры (в генераторе/анализаторе), F1 — окно «О программе».

В режиме генератора задаётся форма сигнала и параметры, при необходимости выбирается фильтр; после применения отображаются осциллограмма, спектр и спектрограмма, доступно воспроизведение и сохранение в WAV. В режиме анализатора загружается файл, выполняется фильтрация (при необходимости), те же визуализации и сохранение результата. В режиме реального времени включается захват с микрофона, отображаются осциллограмма и спектральные характеристики последних 5 секунд.
